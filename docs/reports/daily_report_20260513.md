# 日报 2026-05-13

## 今日完成

### 1. P0 功能收尾

补齐了 MVP 阶段剩余的三个功能点：

- **TPO 场景推荐**：推荐流程新增场合选择（日常 / 工作 / 约会 / 旅行 / 运动 / 派对），与天气温度做双重过滤，同时结合用户风格偏好排序
- **启动主动推送**：App 启动时自动查询天气，有衣橱数据时主动询问是否要看今日穿搭建议
- **上传图片质量引导**：上传衣服前展示拍摄贴士（平铺白底 / 均匀光线 / 完整拍摄）

---

### 2. AI 硬件特性接入

基于设备摄像头 + 麦克风 + GPS 的硬件原生能力，实现了以下纯 App 做不到的功能：

**GPS 定位**
- 天气推荐不再需要用户手动输入城市，由设备定位自动获取

**麦克风语音识别**
- TPO 场合选择支持语音输入，如直接说「约会」「上班」自动解析
- 温感反馈支持自然语言，如说「有点冷」「热死了」自动更新偏好记忆
- 城市名支持中文输入，自动映射英文

**24/7 环境音温感监听**
- 麦克风后台常驻，捕捉到「好冷啊」「热死了」等关键词时自动触发
- 今日已有穿搭记录 → 直接更新温感偏移，无需用户任何操作
- 今日无穿搭记录 → 询问用户今天穿了什么，建立上下文再调整

### 5. 竞品文档更新（taobao_vs_ours.md）

- 更新了与 AICloset 的对比表格，将已实现功能状态从 ❌/⚠️ 刷新为 ✅
- 新增**「④ AI 硬件原生被动感知层」**章节，6 行对比表说明摄像头/麦克风/GPS 在淘宝和 AICloset 上均不可实现
- 核心结论：纯 App 数据飞轮需要用户持续主动操作；硬件数据飞轮被动积累，随时间差距拉大

---

### 6. Bug 修复

- **SQLite 列顺序错位**：`get_user_profile` 使用 `SELECT *` 时，ALTER TABLE 新增列顺序与 cols 列表不一致，导致 `temp_offset` 读到 `upload_time` 的值。修复方式：改为显式指定列名查询
- **类型错误**：`temp_offset` 从数据库读出为字符串，加 `int()` 强转修复
