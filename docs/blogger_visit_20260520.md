# 博主到访准备 — 2026-05-20

> **目标：** 用博主的真实衣橱完成一次端到端产品演示，让她感受到"我的衣服被 AI 整理、推荐、试穿、沉淀成 Looks"的完整体验，同时收集产品反馈、数据授权和后续合作素材。

---

## 一、核心演示目标

这次不是单纯展示技术，而是让她体验到一个完整闭环：

```text
真实衣橱入库
→ 分类浏览
→ 今日穿搭推荐
→ 单品替换
→ 保存 OOTD 到 Looks
→ 随手试穿任意衣服
→ 全身照识别 / 入库 / OOTD 记录
```

我们希望她现场产生三个感受：

1. **"这是我的衣橱"** — 她能看到自己的衣服被分类整理成数字衣橱，而不是一个通用 demo。
2. **"它真的能帮我搭配"** — 今日推荐不是随机生成，而是基于天气、衣橱单品、风格标签组合出来的。
3. **"以后我可以持续用它"** — 她可以保存 Looks、记录 OOTD、试穿新衣服，未来还能服务内容创作。

核心一句话：

> "我们想让你的真实衣橱变成一个每天能帮你做决定、记录穿搭、辅助创作的 AI 衣橱。"

---

## 二、演示流程（30-40 分钟）

### 开场（1-2 min）

**话术：**

> "我们这次不是给你看一个普通 demo，而是把你发给我们的衣服照片处理成了一个真实的数字衣橱。今天想让你像真实用户一样试一下：看衣橱、看推荐、替换单品、保存穿搭、试穿衣服。我们也想听你觉得哪里准、哪里不准、哪里真的有用。"

---

### 第一段：看自己的数字衣橱（8-10 min）

**展示目标：** 让她先建立"这是我的衣柜"的感觉，先让她滑、点、看，不要急着讲算法。

**展示内容：**

1. 衣橱总览页 — 按品类 Tab 分类：上装 / 下装 / 外套 / 鞋履 / 配件 / 全身
2. 让她自己滑动几个 Tab，观察她会不会自然停留、点开、纠错
3. 单品详情弹窗 — 白底图 + 自动标签（品类、颜色、风格、季节、厚度、版型、描述）

**话术：**

> "这些都是从你发来的照片里识别出来的，按品类放进衣橱，你可以像翻自己的衣柜一样滑。"
>
> "这里每件衣服都有 AI 自动打的标签，你可以帮我们看看哪些准，哪些不准。"

**现场收集：**

- 有没有明显识别错的单品？
- 有没有分类放错的衣服？
- 白底图是否能认出原来的衣服？
- 她觉得哪些标签有用，哪些没必要？
- 有没有她记得有、但衣橱里没出现的衣服？

---

### 第二段：今日穿搭推荐（8-10 min）

**展示目标：** 展示从"衣橱管理"进入"每天帮我穿什么"的核心使用场景。

**展示内容：**

1. 首页今日推荐 — 基于今天天气 + 她的衣橱推荐 4 套搭配（含推荐理由、风格标签）
2. 点开一套，展示单品组合：上装 + 下装 + 鞋 + 外套/配件
3. 单品替换 — 点击替换按钮，从衣橱同类单品中换掉一件，展示替换后效果
4. 保存 OOTD 到 Looks — 选中一套，点保存，该搭配进入 Looks 记录页

   > **这是重要闭环：推荐不是看完就结束，而是可以沉淀成她自己的穿搭日志。**

**话术：**

> "这几套是根据今天的天气和你的衣橱自动搭出来的。你可以直接选一套，也可以觉得某一件不想穿，然后换成你衣橱里的另一件。"
>
> "如果你觉得这套今天真的会穿，我们可以把它保存成一条 OOTD，之后在 Looks 里就能回看。"

**引导她参与：**

- "这四套里你最可能穿哪套？"
- "哪套你一定不会穿，为什么？"
- "你替换单品时更看重颜色、风格、还是显瘦？"
- "保存 OOTD 这个动作你觉得自然吗？"

---

### 第三段：随手试穿（8-10 min）

**展示目标：** 展示"想试什么就试什么"的能力，不只依赖推荐流程。

**展示两类试穿：**

**A. 试穿衣橱里的任意衣服**

从她已入库的衣橱里选一件，生成穿在她身上的效果。
适合选：她刚才感兴趣的单品、她说"不太会搭"的单品、风格比较明显的单品。

**B. 试穿一张新的衣服照片**

现场拿一张新衣服照片（或现场拍），展示也能进入试穿流程。

功能价值：

```text
看到一件新衣服
→ 拍照 / 上传
→ 看穿在自己身上的效果
→ 决定要不要入库 / 搭配 / 购买
```

**话术：**

> "除了推荐，我们也想做一个更随手的试穿。你可以从衣橱里任意选一件，或者给我们一张新衣服照片，直接看它穿在你身上的大概效果。"
>
> "这个功能现在还在迭代，所以我们不会把它包装成完全真实的最终效果。现阶段主要是帮你快速判断风格、颜色和整体感觉适不适合。"

**现场收集：**

- 她觉得试穿图最影响判断的是什么（脸？身材比例？衣服版型？颜色？）
- 她更想试：自己衣橱里的衣服 / 新看到的衣服 / 小红书种草图里的衣服？
- 她觉得这种试穿能不能帮助她减少纠结？

---

### 第四段：全身照识别与 OOTD 入库（5-8 min）

**展示目标：** 展示 OOTD 记录的入口 — 用户拍一张当天穿搭，系统识别单品、生成穿搭记录。

> **注意定位：这不是规划未来穿什么，而是记录今天穿了什么。**

**展示流程：**

1. 现场拍一张她的全身照（背景干净、自然站姿）
2. 系统识别她身上的衣服（上装 / 下装 / 鞋 / 外套 / 配件）
3. 识别到的新单品可一键加入衣橱
4. 同时生成今天这套的 OOTD Look，保存到 Looks 页

**当前状态说明：**

> 当前版本可以展示"拍全身照 → 识别衣服 → 加入衣橱"的方向；"自动保存成一条 OOTD Look"是下一步应实现的关键闭环，目前暂未完全实现。

**话术：**

> "我们希望以后你每天不用手动记穿了什么，只要拍一张全身照，系统就能识别这套用了哪些衣服，并保存成一条 Look。"
>
> "现在这一版还没有把 OOTD 自动保存闭环完全打通，但这是我们下一步最想做完整的地方。"

---

### 第五段：Looks / OOTD 沉淀（3-5 min）

**展示目标：** 让她理解 Looks 不是普通相册，而是"穿搭记录 + 单品关系 + 内容资产"。

**展示内容：**

1. 刚才从推荐保存的一套 OOTD
2. Look 详情页：当天穿搭图 + 包含的单品 + 风格标签 + 日期
3. 未来可以回看：最近穿过什么 / 哪些衣服很少穿 / 哪些组合经常出现

**话术：**

> "我们不希望 Looks 只是照片墙，而是每套穿搭都知道它由哪些单品组成。这样之后才能做重复提醒、穿着频次、内容整理，甚至一键生成穿搭贴纸。"

## 四、产品视角需要验证的问题

### 1. 衣橱管理是否有吸引力？

- "你看到自己的衣服被这样分类，会想继续翻吗？"
- "如果这个衣橱是你日常用的，你最常点哪个分类？"
- "你觉得白底图重要吗，还是原图也可以？"

### 2. 推荐是否真的像她？

- "这几套里哪套最像你？"
- "哪套你完全不会穿？"
- "如果让你改，你第一件会换什么？"
- "你平时搭配时先选上衣、下装，还是先看场合？"

### 3. 替换单品是否是刚需？

- "你看到一套推荐时，会想直接接受，还是一定会想换一两件？"
- "替换时你希望系统给你推荐相似的，还是给你更多不同风格的？"
- "你更想换颜色、款式，还是厚度？"

### 4. 保存 OOTD 是否自然？

- "如果今天真的穿了这套，你会愿意保存吗？"
- "你会回看以前穿过什么吗？"
- "你会在意最近有没有重复穿同一套吗？"

### 5. 随手试穿的价值在哪里？

- "你什么时候最需要试穿？买衣服前、出门前，还是拍内容前？"
- "如果在店里看到一件衣服，拍一下就能试穿，你会用吗？"
- "如果小红书看到一件类似的衣服，你会想试在自己身上吗？"

## 五、延伸调研（demo 结束后轻松聊）

> 不要在演示流程中途全部问，选她最有反应的方向深入。

### A. OOTD 穿搭日志

- "你现在会记录每天穿什么吗？手机相册、小红书草稿、还是根本不记？"
- "有没有过想重复一套但想不起当时怎么搭的情况？"
- "你觉得'穿搭频次统计'有意思吗——比如这件今年穿了 12 次，那件只穿了 1 次？"

产品判断：如果她有回看 / 复盘 / 避免重复需求 → Looks 和 OOTD 日志值得优先做。

### B. 内容创作辅助

- "你发 OOTD 从拍照到发出去，最麻烦的是哪一步？"
- "粉丝问链接或单品信息时，你现在怎么整理？"
- "如果系统自动生成这套搭配用了哪些单品，你会用来辅助发内容吗？"
- "你会需要一键导出穿搭贴纸 / 单品平铺图吗？"

产品判断：如果她把它当内容素材管理工具 → 博主版可以强调 Looks、单品卡片、导出、贴纸。

### C. 价格与穿着频次

- "如果每件衣服可以填价格，你会愿意填吗？"
- "你会不会用这个来清理衣柜，看哪件衣服最不值？"

产品判断：如果她对 cost per wear 有兴趣 → 价格字段有价值，否则先做可选。

### D. 场合与社交关系

- "你记穿搭时会不会想顺手记一下'这套穿去了哪里'？"
- "有没有'这群人面前不能再穿这套了'的情况？"

产品判断：如果她对场合记忆敏感 → OOTD 需要加 occasion / event tag。

### E. 逛街扫橱窗

- "你逛街时有没有看到一件衣服但不确定自己穿上好不好的经历？"
- "如果对着橱窗拍一张照片，就能看到穿在你身上的效果图，你觉得有用吗？"

产品判断：如果她反应强 → 随手试穿可以成为硬件/移动端的重要场景。

### F. 单品价格与标签

- "你觉得哪些标签没用？缺了什么标签？"（我们现有：品类、颜色、风格、季节、厚度、版型）
- "如果每件衣服可以填价格，你最想用来做什么？"

### G. OOTD 转 Sticker

- "你做 vlog 时，有没有想在画面里标注'今天穿了什么'的需求？"
- "如果可以一键把今天的搭配导出成贴纸（白底单品平铺图 or 穿在身上的效果图），你会用在哪里？"

## 九、备用方案

**如果 2000 张批跑来不及：**

> "我们先用已经处理完成的一部分衣服做 demo，剩下的还在后台继续处理。今天主要想让你体验完整流程。"

重点演示衣橱分类 → 推荐 → 替换 → 保存 Look → 试穿，而不是强调"2000 件完整度"。

**如果推荐效果不好：**

> "这正好是我们要验证的地方。现在推荐还主要依赖标签和天气，你觉得不准的地方会直接帮助我们调整偏好模型。"

然后让她手动替换单品，展示交互价值。

**如果试穿效果不好：**

> "试穿目前是体验判断，不是最终成片。我们现在更想验证你会不会在这个场景下用它，以及什么样的效果你才觉得可信。"

**如果现场接口慢：** 使用提前准备好的截图 / 本地案例继续讲流程。
