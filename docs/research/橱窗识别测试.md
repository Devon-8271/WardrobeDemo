### 评估

**识别准确度**

| 单品 | 店内是否可见 | 备注 |
|---|---|---|
| 蓝色衬衫 | ✅ | 右侧模特有 |
| 卡其拼色外套 | ✅ | 左侧模特有 |
| 海军蓝T恤 | ✅ | 衣架大量存货 |
| 白色蛋糕裙/长裙 | ✅ | 可见 |
| 粉色格纹衬衫 | ⚠️ | 疑似，不确定是否精确匹配 |
| 配件（帽/鞋/袜） | ❌ | 储物格内可见但未被利用 |
| 裤架单品（格纹/条纹） | ❌ | 大量存货但均未出现 |

**问题小结**

- 主体大件（衬衫/外套/T恤）识别准确
- 配件识别率低（帽子、袜子、鞋未被采用）
- 叠挂衣架识别易漏：裤架上数十件裤子几乎全部忽略
- 模型可能在"泛化风格"而非真正识别具体单品——无法区分"这家店的粉色衬衫"和任意粉色衬衫
- 人物特征未锁定：若模型换脸/换体型则识别失效

**结论**：主装可用，配件/下装弱；需改 prompt 提升单品来源约束和人物锁定。

## V2 — 改进 prompt（待测）

### Prompt（`store_outfit_v1.json`）

```
以提供的参考图为基础：首张为目标人物全身照，其余为服装门店实拍。
请为该人物生成 2×2 穿搭效果图，共 4 格，每格一套完整搭配。

【人物】严格保留首张参考图中的脸部、发型、肤色、体型特征，仅替换服装，四格为同一人。
【服装】每套上装和下装均来自门店实拍图中可识别的真实单品，搭配完整，可加配件（帽子/包/鞋）。
【风格】四套风格各异，体现不同场景：街头休闲、学院通勤、约会出行、活动穿搭。
【画面】背景统一为简洁室内浅色环境；四格等大，无边框，无文字，无水印。
```

**改进点**

| 维度 | V1 | V2 |
|---|---|---|
| 人物锁定 | 无 | 明确保留脸/发型/肤色/体型 |
| 服装来源 | "这家店"模糊 | 指定从门店实拍图中取可识别单品 |
| 搭配完整度 | 无要求 | 上装+下装+可选配件 |
| 风格分工 | 无 | 4格各占一个场景 |
| 画面规格 | 无 | 统一背景+等大四格+no text |
| 负面词 | 无 | deformed face / blurry / extra limbs |

**传图建议**

- 第 1 张：人物全身参考照
- 第 2–6 张：选覆盖不同区域的门店图（全景 + 上装区 + 下装区 + 配件区 + 模特正面），避免多张同角度衣架侧面

### 评估

**V2 vs V1 对比**

| 维度 | V1 | V2 |
|---|---|---|
| 人物一致性 | ⚠️ 四格人物相似但不完全同一 | ✅✅ 脸部/发型/体型四格高度一致 |
| 背景 | ❌ 暗色编辑风/户外，各格不统一 | ✅ 白色简洁室内，统一 |
| 文字标注 | ✅ 含风格名 + 单品说明（中文） | ❌ 无文字，需额外步骤补描述 |
| 画面干净度 | ❌ 含参考店图拼贴、文字水印 | ✅ 纯效果图，可直接用于展示 |
| 配件利用率 | ⚠️ 有包，无帽/鞋细节 | ⚠️ 右下有包，其余配件弱 |
| 下装多样性 | ✅ 牛仔裤/连衣裙/短裙/长裙 | ⚠️ 偏裙装，裤类只有左上一套 |
| 服装来源可验证性 | ✅ 蓝衬衫/卡其外套/T恤可在店内确认 | ⚠️ 蓝衬衫✅，黑夹克⚠️，棕色套装❌难以确认 |

**V2 最大进步：人物锁定生效**

prompt 里"严格保留脸部/发型/肤色/体型"的约束显著起效——四格明显是同一个人，V1 人物前后脸部有漂移。这是对产品最关键的改进，上身效果图要求人物不能换脸。

**V2 仍存在的问题**

1. **无文字标注**：干净但信息缺失，前端需要额外 LLM 调用补风格描述
2. **棕色套装来源存疑**：店内实拍中深棕色上下搭配不易找到对应单品，可能是模型泛化出来的
3. **配件弱**：帽子/袜子/鞋（储物格里大量存货）仍几乎未被采用
4. **下装偏裙**：4套中 3 套是裙，裤装多样性不足

**核心洞察**

prompt 改进对"画面规格"和"人物锁定"的效果立竿见影；但"从门店实拍中取真实单品"这一约束的执行力仍然有限——模型更可能是在识别整体风格后泛化生成，而非逐件对照店内库存。要真正做到"用这家店的衣服"，可能需要**两步链路**：先用 LLM/VLM 从橱窗图提取单品文字列表，再把列表喂给 image2 生图。

### 评估

**四版横向对比**

| 维度 | V1 | V2 | V3 | V4 |
|---|---|---|---|---|
| 人物一致性 | ⚠️ | ✅✅ | ✅✅ | ✅✅ |

| 画面质量 | ⚠️ | ✅ | ✅ | ✅ |

**V4 进步**

- 二次追问让全身装品类首次进入 catalog，右下格不再是完全找不到来源的泛化套装
- 配件连续两版被采用（白色帽子 + 红色运动鞋）
- 右上、左下两格单品来源完全可验证（100%）

**V4 仍存在的问题**

1. **左上格纹马甲是泛化**：catalog 里无马甲，负向约束"不得自行创作"对上装凑不齐的情况没起效——模型选择自补而非跳过
2. **白色吊带裙近似替换**：catalog 有"白蓝卫衣裙"，模型做了近似推断，生成了款式不同的白色长裙
3. **背景不符合 prompt**：要求简洁白色室内，实际生成植物+灰墙背景（4格一致但不是要求样式）

**核心洞察**

负向约束对"有同品类但没有完全一致款式"的情况不起效——模型会做最近邻替换而非跳过。真正需要的约束逻辑是：**上装凑不齐时，降级用全身装代替，而非自行创作**。
