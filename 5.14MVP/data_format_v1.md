# 衣橱数据交付格式 v1.0

**更新日期**：2026-05-14

每件单品为一个 JSON 对象，批量交付为 JSON 数组，文件命名 `items.json`。

---

## 字段定义

| 字段 | 类型 | 说明 |
|---|---|---|
| `category` | string | 大类，固定枚举，见下 |
| `type` | string | 细分类型，固定枚举，无法映射填 `"其他"` |
| `raw_type` | string | 原始识别名称，原样填写，不做标准化 |
| `color` | array | 主色，1-3个，如 `["白色", "米色"]` |
| `style` | array | 风格标签，如 `["休闲", "法式"]` |
| `season` | array | 适穿季节，如 `["春", "夏"]` |
| `warmth` | string | 厚薄程度，固定枚举，见下 |
| `fit` | string | 版型，固定枚举，见下 |
| `material` | array | 面料，如 `["棉", "聚酯纤维"]`，无法判断填 `["不适用"]` |
| `description` | string | 一句话描述，15-30字 |
| `image` | string | 图片文件名，如 `"001.jpg"`，图片统一放在 `images/` 文件夹下 |

---

## 枚举值

### category（必须是以下之一）
`上装` / `下装` / `全身` / `外套` / `鞋履` / `配件`

### type（尽量映射，否则填"其他"）

| category | 允许的 type 值 |
|---|---|
| 上装 | T恤 / 衬衫 / 针织衫 / 卫衣 / 背心 / 吊带 / 马甲 |
| 下装 | 长裤 / 短裤 / 半身裙 / 裤裙 |
| 全身 | 连衣裙 / 套装 / 连体裤 / 连体裙 |
| 外套 | 风衣 / 夹克 / 大衣 / 棉服 / 羽绒服 / 西装 / 皮衣 |
| 鞋履 | 运动鞋 / 高跟鞋 / 靴子 / 凉鞋 / 平底鞋 / 乐福鞋 |
| 配件 | 包 / 帽子 / 围巾 / 腰带 / 首饰 / 其他 |

### warmth（必须是以下之一）
`薄` / `中等` / `厚` / `不适用` / `无法判断`

### fit（必须是以下之一）
`修身` / `常规` / `宽松` / `oversize` / `不适用` / `无法判断`

---

## 示例

### 标准单品

```json
{
  "category":    "上装",
  "type":        "衬衫",
  "raw_type":    "白色泡泡袖衬衫",
  "color":       ["白色"],
  "style":       ["法式", "优雅"],
  "season":      ["春", "秋"],
  "warmth":      "薄",
  "fit":         "宽松",
  "material":    ["棉"],
  "description": "白色泡泡袖衬衫，翻领，宽松廓形，纯棉面料",
  "image":       "001.jpg"
}
```

### 非标准单品（type 无法映射）

```json
{
  "category":    "配件",
  "type":        "其他",
  "raw_type":    "屁帘",
  "color":       ["黑色"],
  "style":       ["街头"],
  "season":      ["春", "秋"],
  "warmth":      "不适用",
  "fit":         "不适用",
  "material":    ["不适用"],
  "description": "黑色流苏屁帘，腰间系带款",
  "image":       "088.jpg"
}
```

---

## 交付说明

1. JSON 文件命名：`items.json`，UTF-8 编码
2. 图片统一放在 `images/` 文件夹，`image` 字段填文件名
3. 无法判断的字段填空字符串 `""`，不要省略字段
